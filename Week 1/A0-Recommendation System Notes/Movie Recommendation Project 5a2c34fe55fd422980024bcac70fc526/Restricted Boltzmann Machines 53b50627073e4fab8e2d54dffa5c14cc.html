Restricted Boltzmann Machines

  • has been in use since 2007
  • Simplest of neural networks; two layers, a visible layer and a hidden layer
  • Train by feeding our training data into visible layer in a forward pass, and training weights and biases between then during backpropagation
  • ReLU as activation function
  • Called restricted because neurons in the same network cant communicate with each other (back then this was something new)
  • Trained by doing forward pass, then a backward pass, where the inputs get reconstructed.
  • Run for multiple epochs

Backward Pass

  • During backpass, we are trying to reconstruct the original input by feeding back the output of the forward pass back through the hidden layer, and seeing what values we end up with out of the visible layer.
  • Since weights are initially random, big difference seen between inputs we started with and the ones we reconstruct.
  • We end up with another set of bias term, this time on the visible layer.
  • Share weights between both forward and back pass but we have two sets of biases - hidden bias that’s used in the forward pass, visible bias used in his backward pass.
  • Main difference from other NNs in that we read output of RBM on the lower level during backward pass as opposed to taking outputs on the other side.

RBM for Recommendation

  • The input is rating of users for all movies
  • We’re trying to learn weights and biases to let us reconstruct ratings for user/movie pairs we don’t know yet.
  • Rating are categorical data: if user rates a movie 5 start, the input would be [ 0 0 0 0 1 ]
  • When we’re done training the RBM, we’ll have set of weights and biases that should allows us to reconstruct ratings for any user.
  • For new user, we run it once again using the known rating of the user we’re interested in, run through those in the forward pass, then back again in the backward pass, to end up with reconstructed rating values for that user. Run softmax to transform group of 5 rating values to translate the output back into 5-star rating for every item.

Problems

  • The data is sparse
  • If we’re training on every possible combination of user and movies, most of that data will be missing.
  • We need to predict user ratings for every movie though, so we need to leave space for all of them.
  • N movies → N * 5 visible nodes
  • We deal with this by excluding any missing ratings from processing while we’re training the RBM.
